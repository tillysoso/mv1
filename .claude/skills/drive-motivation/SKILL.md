# Drive Motivation Framework

Framework for designing motivation systems based on what actually motivates humans. Replaces carrot-and-stick thinking with intrinsic motivation (AMP: Autonomy, Mastery, Purpose).

## Core Principle

"The secret to high performance isn't rewards and punishment — it's the deeply human need to direct our own lives, learn and create new things, and do better for ourselves and our world."

For any task requiring cognitive effort, external rewards either don't work or actively hurt performance. Intrinsic motivation drives lasting engagement.

## Scoring

**Goal: 10/10.** Evaluate motivation systems 0-10 based on AMP principles. A 10/10 supports autonomy, enables mastery, and connects to purpose. Always provide current score and improvements.

## The Three Pillars

### 1. Autonomy — Direct our own lives

The desire to have choice over what, when, how, and who we work with.

**The Four T's:**

| Dimension | Question | Example |
|-----------|----------|---------|
| **Task** | What do I work on? | Google's 20% time |
| **Time** | When do I work? | Flexible hours |
| **Technique** | How do I do it? | Choose your own tools |
| **Team** | Who do I work with? | Self-forming teams |

**Product autonomy design:**

| Context | Autonomy Killer | Autonomy Enabler |
|---------|----------------|------------------|
| Onboarding | Forced linear tutorial | Choose your own path |
| Customization | One-size-fits-all | Themes, layouts, preferences |
| Workflow | Rigid process | Custom automations, flexible steps |

**Warning signs:** "You must complete X before Y", forced tutorials, mandatory notifications, no customization.

### 2. Mastery — Get better at something that matters

**Three Laws of Mastery:**
1. **Mindset**: Growth mindset — ability is developed, not fixed. Frame failures as learning.
2. **Pain**: Requires deliberate practice. Challenge must match skill (Flow state).
3. **Asymptotic**: You can approach mastery but never fully arrive. The pursuit is the reward.

**The Flow Channel:**
- Too easy → boredom
- Too hard → anxiety
- Just right (challenge ≈ skill) → Flow

**Product mastery design:**

| Context | Mastery Design |
|---------|---------------|
| Progress | Visible skill development (GitHub graph, Duolingo levels) |
| Difficulty | Adaptive challenge matching user skill |
| Feedback | Immediate, clear signals (Grammarly real-time) |
| Goals | Clear, achievable milestones (LinkedIn profile strength) |

**Warning signs:** No way to see improvement, same difficulty for everyone, delayed feedback.

### 3. Purpose — Serve something larger than ourselves

**Three expressions:**
- **Goals**: Purpose-driven objectives ("With every purchase, TOMS helps someone in need")
- **Words**: Language of purpose ("associates" not "employees")
- **Policies**: Actions that demonstrate values (Patagonia's "Don't Buy This Jacket")

**Product purpose design:**

| Context | Purpose Design |
|---------|---------------|
| Impact | Show user's contribution (Wikipedia edit count) |
| Community | Connect to something bigger (open source) |
| Mission | Clear inspiring why ("Organize the world's information") |

## Gamification Done Right

| Principle | Wrong (Extrinsic) | Right (Intrinsic) |
|-----------|-------------------|-------------------|
| Autonomy | Forced challenges | Opt-in, choose challenges |
| Mastery | Points for everything | Skill-based progression |
| Purpose | Pointless leaderboards | Contribute to community |

**Caution on streaks**: Can shift from mastery (intrinsic) to loss aversion (extrinsic).

## Compensation

1. Pay enough to take money off the table
2. Then focus on AMP — beyond "enough," more money ≠ more motivation
3. Use "now-that" rewards (unexpected recognition) not "if-then" rewards (contingent bonuses)

## Type I vs. Type X

| Type X (Extrinsic) | Type I (Intrinsic) |
|--------------------|---------------------|
| External rewards driven | Autonomy, mastery, purpose driven |
| Short-term focused | Long-term focused |
| Fixed mindset | Growth mindset |

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Points for everything | Reserve for meaningful milestones |
| Mandatory participation | Make engagement opt-in |
| Same challenge for everyone | Adaptive difficulty |
| No visible progress | Progress indicators, skill tracking |
| "If-then" bonuses | Pay fairly, focus on AMP |

## Quick Diagnostic

| Question | If No → Action |
|----------|----------------|
| Can users choose what/when/how? | Add choices, flexibility, customization |
| Can users see their progress? | Add progress tracking, skill levels |
| Is challenge matched to skill? | Implement adaptive difficulty |
| Is there immediate feedback? | Add real-time response |
| Does the user know WHY this matters? | Connect to mission, show impact |
| Using "if-then" rewards? | Switch to "now-that" or intrinsic design |
